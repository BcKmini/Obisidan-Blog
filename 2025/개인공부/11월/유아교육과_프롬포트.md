<AI 챗봇형 안내장 제작 프롬프트>

나는 ‘맘토리(Momstory)’라는 이름의 유치원 안내 전용 대화형 챗봇 설계자이다.
‘맘토리’는 mom + story의 의미를 가진, 부모의 궁금증을 해결해 주는 따뜻한 친구 같은 도우미이다.
내가 만들고 싶은 것은 문자 입력 버튼 없이, 버튼 클릭만으로 정보를 주는 챗봇형 안내장이다.
아래 요구사항을 바탕으로 Q(ACTION 버튼) 구성, A(답변), 전체 화면 구성, 톤과 스타일 가이드를 완성된 형태로 만들어줘.

챗봇 개념 및 동작 방식
부모가 직접 챗봇에게 말을 입력하지 않는 무채팅 챗봇
원하는 정보의 Q 버튼 선택하고 채팅창에 뜨면 A 답변이 그 아래 채팅으로 표시
Q는 스마트폰 화면 아래쪽에 가로 스크롤 가능한 버튼 배열로 나열
: 화면에 A가 표시
: A 문장을 자동 음성으로 읽어줌
: 단, 이미지는 읽지 않음 / 텍스트만 읽음

2. 대상 이용자
유치원생(만 3·4·5세)의 학부모 전체
스마트폰을 사용하기 때문에 모바일 중심으로 설계

3. 디자인 가이드
4) 배경 및 색상
따뜻한 파스텔 톤
완전한 화이트가 아닌 아이보리색 배경
포인트 컬러 : 화이트, 그린 계열
5) 폰트
간결한 둥근 고딕 계열
너무 얇거나 굵지 않게
모바일에서 선명하게 읽히는 두께
6) 그래픽 요소
손으로 그린 그림의 느낌 
: 말풍선
소통을 상징하는 요소
전체적으로 귀엽고 부드러운 톤
7) Q 버튼 스타일
라운드형 모서리 사각형
테두리 없음
버튼 앞에 Q 내용의 상징적 이미지 사용
가로 스크롤로 좌우 이동 가능
탭 시 자연스럽게 눌리는 애니메이션
8) A 답변 영역
상단에 사진을 넣을 자리 확보
이미지는 실제로 나타나지만, AI가 읽지 않음
텍스트는 표, 기호, 이미지, 단락 구분 등을 활용해서 가독성 향상

9. 대화 톤과 문체
존댓말과 약간의 반말이 섞인 부드럽고 친근한 톤
교사가 부모와 자연스럽게 대화하는 느낌
따뜻한 말투
아이를 사랑하는 교사 같은 친절함
안정감을 주는 말투
너무 유아화 되지 않도록 부모 기준의 자연스러움 유지
너무 딱딱한 공지문 어투 금지
음성 읽기 기능을 고려하여 문장이 너무 길지 않고, 리듬감 있게 구성

5. 시작 인사말

13. 최종 요청 문장
위의 모든 요구사항을 충족하는 챗봇형 안내장을 만들어 주되  UI 구조, 텍스트 예시, 버튼 배열 방식, 음성 읽기 방식까지 포함해 정교하게 제작해줘. 그리고 유치원에서 사용하는 것인 만큼 부드러운 인상을 줄 수 있는 디자인으로 구성하되, 가독성이 떨어지지 않는 구성으로 디자인해 줘. 추가로 이미지는 내가 같은 경로에 디렉토리 폴더에 이미지를 후에 넣을거며 Q에 답변에 대한 이미지는 행사 날짜, 행사 시간, 준비물, 드레스코드, 주차 공간, 행사 타임테이블을 넣을거고 행사 장소는 kakaomap api를 사용해 충주 호암체육관에 장소를 맵으로 찍어줄거야 답변에 같이
